CPU负载

linux
|浏览量:

Linux操作系统，会记录CPU负载，以供调度器进行负载均衡。还会让其他子系统参考这些CPU负载来进行相应调整，例如像DVFS等。

CPU负载分为三种：

  • 全局负载
  • 运行队列负载
  • PELT（Per Entiry Load Tracking）

指数加权移动平均（EWMA）

这里先看一下所谓的指数加权移动平均是什么，因为下面在计算CPU的全局负载/PELT的时候都会用到这种方式来计算。

指数加权移动平均exponentially weighted moving average (EWMA)，又叫指数移动平均exponential moving average (EMA)，是计算平均值的一种方式。

全局负载

全局负载定义

这里所谓的CPU负载，就是每隔一段时间统计CPU的runqueue上是否有TASK_UNINTERRUPTIBLE和TASK_UNINTERRUPTIBLE类型的进程个数。
例如一个CPU上每隔5秒统计了1分钟，每次查看的时候CPU上都有一个TASK_UNINTERRUPTIBLE或者TASK_UNINTERRUPTIBLE类型的进程。
那CPU负载统计出来就是1。如果只有一半时间有进程在runqueue上，那统计出来就是0.5，如果每次统计都是2，那统计出来就是2，依次类推。

这里为什么要加上TASK_UNINTERRUPTIBLE类型的？有些人数是TASK_UNINTERRUPTIBLE是等IO操作，所以要加上。这个理由有点牵强~ 后续补充…

全局负载计算

全局负载的计算分两个不分，一个是定时计算per-CPU的瞬时负载加到calc_load_tasks中，还有一个就是定时将这些瞬时负载计算并保存到avnrun变量中。

定时计算per-CPU负载

calc_global_load_tick就是每隔5秒会统计TASK_UNINTERRUPTIBLE和TASK_UNINTERRUPTIBLE两种类型的进程个数，
并与5秒前的进程数进行对比，将多出来的进程加到calc_load_tasks中或是将减少的进程个数从calc_load_tasks中减掉。

运行队列负载

PELT(Per-entity load tracking)

PELT是计算每个调度实体的负载贡献。上述的全局的负载，是每5秒统计的所有CPU上的TASK_UNINTERRUPTIBLE和TASK_UNINTERRUPTIBLE类型的进程的个数。
