通过ll/sc实现

在ARMv6以上（含v6）构架有了多核的CPU。为了在多核之间同步数据和控制并发、ARM在内存访问上增加了独占监测（Exclusive monitors）机制、并增加了相关的指令。
ll(load-link)指令、sc(store-condition)指令。ll对应的arm指令是ldaxr、sc对应的arm指令是stlxr。
进行原子操作的大致流程如下：

  • 使用 LDXR (Load-Exclusive) 指令加载数据到寄存器，同时标记对应的内存地址为独占状态。
  • 在寄存器中进行数据修改。
  • 使用 STXR (Store-Conditional) 指令尝试将修改后的数据写回内存。如果该内存区域在 LDXR 和 STXR 之间没有被其他处理器修改，STXR 将返回成功，并且独占状态会被清除；如果被修改过，STXR 将失败，需要回到第一步重新尝试。

上述函数流程如下：

  • (1) ldrex %0, [%3]：意思是将&v->counter指向的数据放入result中，并且（分别在Local monitor和Global monitor中）设置独占标志。
  • (2) add %0, %0, %4：result = result + i
  • (3) strex %1, %0, [%3]：意思是将result保存到&v->counter指向的内存中，此时 Exclusive monitors会发挥作用，将保存是否成功的标志放入tmp中。
  • (4) teq %1, #0：测试strex是否成功（tmp == 0 ？？）
  • (5) bne 1b：如果发现strex失败，从(1)再次执行。

注意：Arm上原子操作时、ldaxr和stlxr（这两个是ldar/stlr的exclusive版本）已经隐含了barrier操作。所以在做原子操作之后、没有必要手动添加barrier、以免影响性能。

这两个原子指令会将共享变量先load到本核所在的l1-cache中进行修改、在锁进程少的情况下性能较好、但在锁竞争激烈时、会导致系统性能下降。
Arm-v8.1规范中引入了新的原子操作指令扩展LSE（Large System Extensions）、将计算操作放到l3-cache去做、增大数据共享范围、减少cache一致性耗时、在锁竞争基类时可以提升锁的性能。
打开方式是在GCC编译时添加LSE相关选项：GCC6.0以上版本（建议使用gcc7.3.0以上版本）、在编译选项中增加-march=armv8-a+lse，或者-march=armv8.1-a，或者-march=armv8.2-a选项。
