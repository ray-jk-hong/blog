buildroot准备

下载

通过git clone https://github.com/buildroot/buildroot.git下载。

编译

make qemu_riscv64_virt_defconfig
make -j32

编译完成之后，在output/image就有要使用的

busybox准备

https://devpress.csdn.net/linux/62ff50d3c67703293080515d.html

qemu准备

./configure –target-list=riscv64-softmmu
make -j $(nproc)

linux准备

make distclean
make clean
make ARCH=riscv CROSS_COMPILE=riscv64-linux-gnu- defconfig
make ARCH=riscv CROSS_COMPILE=riscv64-linux-gnu- Image -j $(nproc)

opensbi

编译

make PLATFORM=generic PLATFORM_RISCV_XLEN=64 PLATFORM_RISCV_ISA=rv64g_zicbom FW_TEXT_START=0x0 FW_JUMP=y FW_JUMP_ADDR=0x200000 FW_JUMP_FDT_ADDR=0xfe00000 CROSS_COMPILE=riscv64-linux-gnu- distclean

make PLATFORM=generic PLATFORM_RISCV_XLEN=64 PLATFORM_RISCV_ISA=rv64g_zicbom FW_TEXT_START=0x0 FW_JUMP=y FW_JUMP_ADDR=0x200000 FW_JUMP_FDT_ADDR=0xfe00000 CROSS_COMPILE=riscv64-linux-gnu- -j $(nproc)

qemu

编译

./configure –target-list=riscv64-softmmu
make -j $(nproc)

qemu执行

qemu-system-riscv64 -nographic -machine virt -kernel linux/arch/riscv/boot/Image

opensbi调试

社区版本，opensbi加载地址默认为0x80000000。所有在gdb加载opensbi的elf文件的时候需要加偏移。
gdb: add-symbol-file opensbi.elf 0x80000000

加载完之后，可以使用stepi执行到第一行。第一行就是qemu设置的vector代码了，这段代码被放到qemu的mrom段。如下所示：

qemu输出dtb文件

qemu-system-riscv64 -M virt,dumpdtb=virt.dtb
dtc -I dtb -O dts virt.dtb > virt.dts

gdb

add-symbol-file xxx.elf 0x80000000：符号加载，起始地址是 0x80000000
stepi：向下执行一行
info address _start： 查看某个符号地址
info reg：打印通用的寄存器
info all-reg：打印所有的寄存器

x/10i 0x1000： 打印0x1000开始的一段汇编
