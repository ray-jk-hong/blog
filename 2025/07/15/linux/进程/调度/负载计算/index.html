Linux操作系统会记录CPU负载以供调度器进行负载均衡、还会让其他子系统参考这些CPU负载来进行相应调整，例如像DVFS等。

CPU负载分为三种：

  • 全局负载
  • 运行队列负载
  • PELT（Per Entiry Load Tracking）

在看Linux怎么计算上述三种负载之前、我们先看一下指数加权移动平均(EWMA)算法、因为上述几种负载计算中都会用到此类算法。

全局负载

全局负载定义

这里所谓的CPU负载，就是每隔一段时间统计CPU的runqueue上是否有TASK_RUNNING和TASK_UNINTERRUPTIBLE类型的进程个数。
例如一个CPU上每隔5秒统计了1分钟，每次查看的时候CPU上都有一个TASK_RUNNING或者TASK_UNINTERRUPTIBLE类型的进程。
那CPU负载统计出来就是1。如果只有一半时间有进程在runqueue上，那统计出来就是0.5，如果每次统计都是2，那统计出来就是2，依次类推。

这里为什么要加上TASK_UNINTERRUPTIBLE类型的？有些人数是TASK_UNINTERRUPTIBLE是等IO操作，所以要加上。这个理由有点牵强~ 后续补充…

PELT(Per-entity load tracking)

每个调度实体的负载、可以用struct sched_avg来表示。结构体如下：

调度实体包含struct sched_avg：

PELT是以1024us为周期、按照上述的EWMA方式计算调度实体的以下几个时间。

  • 调度实体挂载在rq队列中的时间：就是上述的load_sum（这是总共的时间）和load_avg（这是平均的值）
  • 调度实体runnable状态的时间：查看se_runnable函数、非调度组的调度实体只要挂在rq中就是属于ruannable状态。组调度中的调度实体、返回的值是se->runnable_weight。
  • 调度实体running的时间：表示调度实体实际执行的时间、就是挂到rq队列并执行的时间。（在每1024us时间打点的时候确认的）

查看__update_load_avg_se()函数可知、对于调度实体来说、load/runnable是一样的、都是!!sched_entity->on_rq来判断。!!sched_entity->on_rq的值是1、表示调度实体已经调用enqueue_entity挂到调度队列中了。值是0表示调度实体已经调用dequeue_entity从调度队列中删除了。

跨越1ms的场景

下面来看一下跨越1ms场景下、是怎么累计时间的。如下图所示、在(1)记录完时间、在(2)重新记录时间、中间跨国的时间起码1ms以上：

根据EWMA展开就是D1 * y^P + 1024 * (y^(p - 1) + y^(p -2) + ... + y) + D3。下面根据代码看一下具体怎么算的。

accumulate_sum函数中按照如下方式计算并累加时间：
1.delta += sa->period_contrib：上一次记录时间的时候剩下的时间加上经过的时间
2.periods = delta / 1024：计算经过的周期（1ms为周期）
3.sa->load_sum = decay_load(sa->load_sum, periods)：将上次的load_sum乘以衰减系数赋值给当前的load_sum（decay_load算法查看上面的EWMA、y^32 = 0.5方式衰减）。runnable_avg_yN_inv就是将y^p << 32提前算好保存起来的。因为y^32 = 0.5、y的值是0.9785720621、计算一下可以得到runnable_avg_yN_inv数组的值。decay_load()函数中、超过63就返回0是因为超过63个半衰减周期（半衰减周期就是32ms、因为y^32 = 0.5）、其值已经可以忽略不计了、所以就返回了0。（63 * 32 = 2016 约等于2秒）。
4.contrib = __accumulate_pelt_segments(periods,1024 - sa->period_contrib, delta)：D1乘以n个窗口的衰减（n就是periods）、D2乘以n-1个窗口的衰减、再加上D3得到新的contrib的值。D1和D3的时间都好算、计算D2的式子展开就是1024 * (y^(p-1) + y^(p-2) + ... + y)。其中(y^(p-1) + y^(p-2) + ... + y) = (1 - y^p) / (1 - y)、可以将左边的式子转成

其中

这个算式可以转成(1-y^n) / (1-y)、当n趋近于无穷时就是1 / (1 - y)。所以上面的式子就是 1024 / (1 - y)、算出来的值就是LOAD_AVG_MAX。

这个可以转成 y^p * (1024 / (1 - y))、其中1024 / (1 - y) = LOAD_AVG_MAX、所以就是y^p * LOAD_AVG_MAX。这就是对LOAD_AVG_MAX做p的衰减的、使用decay_load函数就可以算了。
