cpu拓扑

概念

CPU的拓扑结构可以理解为在硬件层面上多个CPU的排列方式, 对于Linux调度和负载均衡而言，拓扑结构影响着进程在CPU之间迁移所要付出的代价。

cpu拓扑相关的几个概念：

  • socket：对应cpu上的插槽
  • core：硬件上实现真正并行计算的组件，例如我们说一个CPU是“四核八线程”，其中每个核就对应一个Core
  • Hyper-Threading：逻辑上的一个执行单元，多个超线程共享一个核，例如“四核八线程”，就是每两个线程共享一个核。由于多个超线程共享一个物理执行单元，因此超线程并不是硬件层面上独立的并行计算单元，只是在OS级别的并发单元

CPU的拓扑结果、可以在/proc/cpuinfo中查到、在下面也会讲到这个。

cpuinfo

在/proc/cpuinfo中、有几个字段需要解释一下。

  • physical id：主板上实际插入的cpu数量，可以数不重复的physical id有几个
  • core id ：当前物理核在其所处CPU中的编号，这个编号不一定连续
  • cpu cores：该逻辑核所处CPU的物理核数。比如此处cpu cores 是4个，那么对应core id 可能是 1、3、4、5
  • siblings ：单个物理CPU的逻辑CPU数。支持超线程的场景、siblings=cpu cores [*2]。

root_domain and sched_domain

  1. 系统默认情况下、所有的CPU属于一个域、实时进程可以在所有的CPU上进行调度。
  2. 把CPU分为两个组、进程只能在响应的域中的CPU上进行调度

refcount: Domain被引用的次数
span: 此domain使用的cpu范围
online: 在线的cpu范围
rto_mask: 哪些cpu处于overload状态
rto_count: 有多少个cpu处于overload状态
