进程内存统计

linux
|浏览量:

在知道怎么统计进程使用的内存之前，需要知道VSS/RSS/PSS/USS这几个概念。一般进程在执行的时候根据内存使用方式，会有以下几种内存类型。

  1. 进程间共享的内存，一般是与其他进程共享的so库加载区域，shmget方式主动共享的。这种是已经被分配的物理内存的了。
  2. 进程内使用的内存，且已经分配物理内存的。
  3. 进程内使用的内存，但没有分配物理地址的。

上述的1，2，3三种类型的内存对应以下的A, B, C三种类型的内存。

VSS/RSS/PSS/USS就是在统计以上三种类型内存大小的。

VSS：VSS = A + B + C
就是进程能够访问的所有虚拟内存大小。

RSS: RSS = A + B
进程实际能访问的物理内存大小（这么描述更准确一些）。

PSS: PSS = B + (A / 进程数)
刚看的时候不知道为什么要提供PSS，但假设有10个进程加载了某个so并共享。那我们是不是将10个进程的PSS值都加起来就能知道整个10个进程一共使用了多少物理内存呢？

USS: USS = B
这个就是进程自己申请申请的物理内存了。

通过以下几种方式可以知道进程的上述统计结果。

smaps节点：

cat /proc/[pid]/smaps
smaps打印分析请参考：https://ray-jk-hong.github.io/blog/2025/03/23/linux/Debug/proc/pid/

smem工具: smem -p <PID>

pmap工具: pmap -x <PID>

ps工具: ps -o pid,cmd,vsize,rss -p <PID>
