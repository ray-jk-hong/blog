在编写一个程序时，会生成多个文件，并且当多个开发人员一起开发时，可能会出现函数或结构体等名称相同的情况。

在这种情况下，如果函数名、结构体名以及变量名相同，就会发生名称冲突，从而导致错误。

为了避免这种冲突，引入了namespace的概念。

简单来说，namespace是给函数、结构体或变量名等赋予一个归属。通过指定归属，可以明确该函数、结构体等属于哪个命名空间，从而避免混淆。

如上代码所示，如果某个程序员和其他程序员都编写了printAll()函数，那么这个程序就会发生冲突。
通过将每个printAll函数用namespace包裹起来，可以解决冲突问题。

这样，通过创建namespace A和namespace B，将各自的函数包裹起来，就可以避免名称冲突。
很简单吧？

namespace [命名空间名称]
{
  函数、结构体、变量、类等
}

通过这种方式，[命名空间名称] 所属的函数、结构体、变量、类等就会被创建。
调用方法将在下面的“2. namespace 元素访问方法”中详细说明。

2. C++ namespace 元素访问方法

namespace 元素的访问方法大致可以分为 3 种，细分则可以分为 5 种。
我将稍微详细地分为 5 种来说明。

1. 使用限定名称（qualified name）访问

命名空间名称::元素

通过输入命名空间名称并使用 “::” 来访问命名空间内部的元素。这种方法称为限定名称访问，是最明确的方式。

虽然代码会变长，编写时也有些繁琐，但可以避免名称冲突，并且能够清楚地知道函数、变量、类属于哪个命名空间。

直接通过示例来确认吧。

如上所示，当存在 namespace A 和 namespace B 时，使用限定名称访问的方式是
A::元素, B::元素，这样就可以访问命名空间中的元素。

A::printAll();
B::printAll();

通过这种方式调用函数即可。

2. 使用全局 using 声明（declaration）访问

using 声明 用于指定命名空间中的某个变量或函数，使得可以在不使用命名空间名称的情况下访问该元素。

using 命名空间名称::元素;

通过这种方式，可以在不使用命名空间名称的情况下访问特定元素。

在这个示例中，通过 using ABC::init，ABC 命名空间中的 init 函数被全局 using 声明了。
这样，在该文件的任何地方都可以直接调用 init() 函数，而不需要使用 ABC::。

init(); 可以直接调用，当然也可以使用 ABC::init(); 来调用。

而 using 声明未涉及的其他变量或函数，如 print() 和 MAX_CNT，则必须使用 ABC:: 来访问。

3. 使用全局 using 指令（directive）访问

与 using 声明不同，using 指令会解除该命名空间中所有元素的访问限制。

using namespace 命名空间名称;

通过这种方式，可以无需使用命名空间名称来访问该命名空间中的所有元素。

在这个示例中，由于使用了 using namespace ABC，main 函数内部可以直接访问 ABC 命名空间中的所有元素，而无需使用 ABC::。

4. 使用函数内部 using 声明（declaration）访问

与全局 using 声明的功能相同，但由于是在函数内部使用 using 声明，因此其作用范围仅限于该函数内部。

函数(…)
{
  //…

  using 命名空间名称::元素;
  // 从这里开始，该元素可以不使用命名空间名称访问（仅限于该函数内部）
  
}

例如，如果只想在 main 函数中使用 using，而在其他函数中不使用，可以使用这种方式。

在 main 函数内部，using QWERTY::go 声明之前，只能通过 QWERTY::go() 来调用 go 函数，但在 using QWERTY::go 声明之后，可以直接使用 go() 来调用。

与全局 using 声明不同，example() 函数内部无法直接调用 go()。

5. 使用函数内部 using 指令（directive）访问

与全局 using 指令的功能相同，但其作用范围仅限于该函数内部。

函数(…)
{
  //…

  using namespace 命名空间名称;
  // 从这里开始，该命名空间的所有元素可以不使用命名空间名称访问（仅限于该函数内部）
  
}

直接通过示例来确认。

在 main 函数内部，using namespace QWERTY; 声明之前，只能通过 QWERTY:: 来访问 QWERTY 命名空间的所有元素，但在 using namespace QWERTY; 声明之后，可以直接使用 go() 和 back() 来调用。

与全局 using 指令不同，example() 函数内部无法直接调用 go() 和 back()。

6. 全局命名空间访问方法（global namespace）

对于属于命名空间的元素，可以通过限定名称访问方法来访问，即使用 命名空间::元素。

那么，如何明确访问全局命名空间中的元素呢？

“::元素” 可以确保访问的是全局命名空间中的元素。

当使用 using 声明或 using 指令时，可能会出现同名元素冲突的情况，这时可以使用 :: 来明确访问全局元素。

当使用 using 指令解除 Car 命名空间的访问限制时，直接调用 drive() 会导致函数名称冲突。
因此，要调用全局命名空间的 drive 函数，必须使用 ::drive()，而要调用 Car 命名空间的 drive 函数，即使使用了 using 指令，也必须使用 Car::drive()。

当元素名称冲突时，using 指令和 using 声明将失去意义，最终必须使用 命名空间名称::元素 的限定名称访问方法。

3. C++ 标准 std 命名空间

C++ 将所有标准元素放在 std 命名空间 中。
因此，通常在 C++ 代码中，会在文件开头包含头文件，然后使用 using namespace std;。

C++ 的标准元素都在 std 命名空间中，因此如果不使用 using 声明，就必须使用 std::cin, std::cout 等方式来访问。

通过以上内容，我们了解了 C++ 的命名空间 namespace。
虽然使用 using 声明和 using 指令可以方便地访问元素，但如果不恰当地使用，可能会导致名称冲突。
最准确的方法是使用 “命名空间::元素” 的限定名称访问方式。
